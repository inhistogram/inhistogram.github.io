inHistogram

 

Interactive histograms using d3.js computed on your local machine or via a cloud instance.

 

by Pablo de Castro, Xavier Valls, Pablo Jiménez, Nitin Agarwal, Daniel Lemon

How is this different to anything else?

In the past, we would use dc.js and/or crossfilter.js which allows you to do all the computations in the web browser;

 

However it is not great if you want to analyze huge amounts of data (e.g. 3 GB of data)

Therefore...

We built an interface on top of tornado (python web server) and do all the computations locally, but show the results on a local site that presents the data using d3.js

We have a set of histograms and we allow users to select a sub-set of the histogram and see how the data changes.

We don’t use a database because we allow users to create their own scripts

We allow you to work on high volume data, locally.

Status of the project

Over the weekend, we created a working prototype to show that our idea is possible.

This is a complex architecture that is built on top of tornado and uses web sockets to communicate with the client browser, and d3.js for the visualization of the histograms.

Currently we have the built the architecture to support Numpy files, however we can easily integrate ROOT.

The inhistogram.py file loads the tornado server, there we have a working interactive front end view that displays the Numpy data as histograms, we will extend this to include any other type of visualization (bar, 3D histogram, tree, graphs, time-series etc.).

A user can select a sub-set or different variables of the histogram from the site, and using web sockets this will update the histograms with the computed data.
