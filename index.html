Tired of exploring data though scripts?

Do it interactively in your browser!

But just browser based solutions are not powerful enough ( dc.js / crossfilter.js)

 

They do not allow you to analyze important and interesting amounts of data (i.e. 3GB of data).

Therefore...

We have got an interface on top of Tornado (using Websockets and HTTP APIs) to do all the computations locally and display the results on the browser making use of d3.js

You act over the GUI on the browser and analysis in rapidly done by powerful data analysis libraries (i.e. numpy and ROOT)

We have a set of histograms which allow the users to select a sub-set of the histogram and see how the data changes.

Creating a database is tiring and not flexible enough for statistical data analyses.

The user can analyse large datasets, as easy as: inhistogram LHCbData.root

Status of the project

Over the weekend, we created a working prototype to show how our idea is possible.

This is a complex architecture built on top of tornado making use of web sockets for communication with the browser and d3.js for visualizing the histograms.

Currently we have built the architecture to support Numpy files, however we can easily integrate ROOT.

The inhistogram.py file loads the tornado server with a user interactive interface which displays the data as histograms, we can extend this to include any other type of visualization (bar, 3D histogram, tree, graphs, time-series etc.).

A user can select a sub-set or different variables of the histogram to see the updated histograms after the computation of the data.
